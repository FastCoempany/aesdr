## How to use this index

- **Adding a deliverable:** Add a row to the appropriate D-table, set status to `shipped (batch N)`, and bump the status summary at top.
- **Updating a deliverable:** Update the D-file itself, not this index. The status column doesn't track minor edits.
- **Pending → shipped:** Update the row, update the status summary count, leave a one-line note in the SESSION_STATE.md "Recent activity" log.
- **Renumbering:** Don't. D-numbers are immutable once shipped.
