# Lockup Files & Composition Guide

The **AESDR × Partner** co-brand mark is the operator credit on
partner-specific surfaces. The actual lockup files are partner-specific
and arrive separately from this page — they're built against your mark,
sized for your registration page and deck, and signed off before
delivery.

## What you'll receive

After your partnership agreement is countersigned and your assets are
prepared (typically 3 business days), you'll receive five files:

1. **Lockup — horizontal SVG.** For registration page header, email
   footers, web placements where width is available.
2. **Lockup — stacked SVG.** For square placements, social profile
   imagery, vertical surfaces.
3. **Lockup — on-cream PNG (high resolution).** For PDF / print contexts
   where SVG isn't supported (the AESDR brand surface is `--cream`,
   `#FAF7F2`).
4. **Lockup — on-white PNG.** Same composition, white background, for
   surfaces that don't render the cream cleanly.
5. **Composition guide PDF.** A 1-page sheet showing your specific
   lockup at three sizes with the clearspace and color rules called out.

The files arrive via email (not as URLs that anyone could discover) with
your partner-specific naming convention — e.g.,
`aesdr-x-acme-lockup-horizontal.svg`. Use the supplied files as-is.

## Public rules (recap from /partners/kit/lockup-usage)

- AESDR mark in `--ink` (`#1A1A1A`).
- Partner mark in your canonical color.
- The `×` separator in `--ink` at matched x-height.
- Minimum 24px clearspace on all sides.
- No "presented by" / "in association with" / "powered by."
- No third logo.
- No proportion edits.

## Where the lockup goes (per surface)

| Surface | Placement | Which file |
|---|---|---|
| Partner-specific registration page | Top-center, above the fold | Horizontal SVG |
| Workshop deck | Top-left of every slide template | Horizontal SVG |
| Partner-specific email | Footer only — never the header | Horizontal SVG (rendered as PNG inline if SVG isn't supported) |
| Square social placements | Centered | Stacked SVG |
| Print / PDF | Per the composition guide | On-cream or on-white PNG depending on background |
| Anywhere else | Ask — we'll build it | (we'll send a new file) |

## Things that trip people up

**Adding a sponsor / platform logo inside the lockup.** Don't. If your
channel requires a "presented by [Platform]" tag, place it in a separate
band, not inside the AESDR × Partner lockup.

**Compressing the SVG.** Use the supplied file directly. Re-saving
through some image tools introduces subtle proportion drift and the
mark looks wrong by week 2.

**Animating the lockup.** Don't animate the lockup itself. If your
placement requires motion (intro card, stinger), animate around it —
the mark sits and stops.

**Cropping for a tight aspect ratio.** Ask for a custom file before you
crop. We can size to almost any aspect; we'd rather build the file than
have you crop ours.

## What to do if a file looks wrong on your surface

1. Confirm you're using the supplied file unmodified.
2. Compare against the composition guide PDF — the example sizes there
   should match how it's rendering on yours.
3. If it still looks wrong, email `partner@aesdr.com` with a screenshot
   of the placement. We'll either fix the file or fix the placement
   guidance, depending on which is off.

## Approval before publish

The first registration page that uses your lockup goes through one
approval round before publish — partner-specific delivery, partner-specific
placement, partner-specific composition. After that first approval,
subsequent uses on the same surface don't need re-approval.

New surfaces (a new email template, a new social placement) that use the
lockup do need an approval round per the
[Approval Workflow](/partners/kit-private/approval-workflow).
