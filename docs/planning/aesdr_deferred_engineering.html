TASK B.01

Stripe Integration — Products & Checkout

Create Stripe products, checkout sessions, and wire payment flow.

Step-by-Step Execution

  1. Create Stripe account and get API keys
    • Sign up at stripe.com
    • Get publishable key and secret key
    • Add STRIPE_SECRET_KEY and NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY to .env.local
    • Add same keys to Vercel environment variables
  2. Create Stripe products
    • Individual tier: $199 one-time payment
    • Team tier: $999 one-time payment (up to 10 seats)
    • Get price IDs for each
  3. Create app/api/checkout/route.ts
    • Accept priceId and tier in request body
    • Create Stripe checkout session with success_url and cancel_url
    • Return session URL for redirect

Done When

  • Stripe test mode checkout completes successfully
  • Success and cancel URLs redirect correctly

Common Mistakes

  • Using live keys in development
  • Hardcoding price IDs instead of using env vars
TASK B.02

Stripe Webhook Endpoint

Handle payment confirmations and grant course access automatically.

Step-by-Step Execution

  1. Create app/api/webhooks/stripe/route.ts
    • Verify webhook signature using STRIPE_WEBHOOK_SECRET
    • Handle checkout.session.completed event
    • Extract customer email and purchased tier
  2. Write webhook DB logic
    • Create purchases table in Supabase (user_id, stripe_session_id, tier, created_at)
    • On successful payment, insert row
    • Grant course access by updating user metadata or access table
  3. Configure webhook in Stripe dashboard
    • Point to https://aesdr.com/api/webhooks/stripe
    • Select checkout.session.completed event
    • Copy webhook signing secret to env vars

Done When

  • Test webhook fires in Stripe CLI and creates DB row
  • User gets course access after payment

Common Mistakes

  • Not verifying webhook signature (security risk)
  • Not handling duplicate webhook deliveries (Stripe retries)
