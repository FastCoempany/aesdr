TASK 01.01

Inject Anti-Persona Warning

Add a clear warning block that repels poor-fit buyers before they waste time or charge back later.

Owner
Founder
Systems / Tools
Landing page HTML, browser preview, your copy doc
Dependencies
Landing page HTML file available locally.
Files / Places You’ll Touch
aesdr-landing-page.html; optional working copy such as aesdr-landing-page-v2.html
Expected Outcome
A visible warning section on the landing page telling the wrong buyer to leave.

Step-by-Step Execution

  1. Create a duplicate of your landing page file so you are not editing the only copy. In File Explorer, right-click the HTML file, choose Copy, then Paste, and rename the duplicate to something obvious like aesdr-landing-page-v2.html.
  2. Open the duplicate in a code editor such as VS Code. If you do not have VS Code open yet, right-click the file and choose Open With, then select VS Code.
  3. Find the hero section or first major content block under the headline. Use Ctrl+F and search for wording already on the page that sounds like the opener. If there is already a warning-like area, use that instead of creating a second one.
  4. Write the warning text in a plain text note first. Keep it brutally clear. Example structure: line 1 says who this is not for, line 2 says what this product will not do, line 3 says who it is for.
  5. Paste that copy into a new HTML block directly under the hero CTA or directly above the pricing section. Use a div with a class name like anti-persona-warning so you can style it once and reuse the style later.
  6. Add CSS for the warning block in the style section or stylesheet: dark background, thin border, short headline, tight spacing, and one accent color. The goal is clarity, not carnival lights.
  7. Save the file, then double-click the HTML file in File Explorer to preview it in the browser. Refresh after each edit until the warning is easy to see without scrolling much.
  8. Read the warning aloud. If it sounds like vague startup perfume, rewrite it. It should sound like a door bouncer, not a life coach.

Done When

  • The warning is visible on the page without hunting for it.
  • The copy clearly says who should not buy.
  • The new block matches the site visually and does not break mobile layout.

Common Mistakes

  • Hiding the warning too far down the page.
  • Using soft language that fails to filter out poor-fit buyers.
  • Adding it as an image instead of editable text.
TASK 01.02

Establish 'Sober Selling' Angle

Replace hustle-bro language with mature, calm, professional positioning.

Owner
Founder
Systems / Tools
Landing page HTML, copy doc, browser preview
Dependencies
Current landing page copy in editable text form.
Files / Places You’ll Touch
Landing page HTML; optional copy audit spreadsheet
Expected Outcome
The site sounds seasoned and sharp instead of cheesy tech-sales LinkedIn sludge.

Step-by-Step Execution

  1. Open the landing page HTML and also open a blank document beside it for copy notes. You can use Google Docs, Notion, or a plain text file.
  2. Read every headline, subheadline, bullet, and button label from top to bottom. Copy each text string into your notes so you can audit them without getting lost in code.
  3. Mark every phrase that sounds like hustle culture, fake motivation, bro-marketing, or overclaiming. Examples: crush quota, dominate, hack your way, skyrocket, unstoppable, ninja, elite secrets.
  4. For each bad phrase, rewrite it in plain adult language. Swap hype for consequence. Example: replace 'crush quota faster' with 'reduce the chaos that keeps you off quota.'
  5. Paste the rewritten copy back into the HTML one block at a time. Save after each section so if you break something, you know exactly where the problem started.
  6. Preview the page after every two or three edits. You are checking that the new copy still fits the design and does not create awkward line breaks or giant text blocks.
  7. Do a final pass only for tone consistency. The hero, curriculum, pricing, and bio should sound like they came from the same person, not four different freelancers in a trench coat.

Done When

  • No major hustle-bro wording remains.
  • The page sounds calm, direct, and credible.
  • Headlines still fit the design without ugly wrapping.

Common Mistakes

  • Replacing hype with dead boring corporate language.
  • Keeping one or two cringe phrases because they were convenient.
  • Changing copy in the browser instead of the source HTML.
TASK 01.03

Map Trigger Event 1: Paycheck Shock

Create ad and page language for reps getting blindsided by clawbacks or weak commission outcomes.

Owner
Founder / Ads Manager
Systems / Tools
Ad copy doc, landing page, creative notes
Dependencies
Audience positioning written down first.
Files / Places You’ll Touch
Ad concept doc; optional campaign naming sheet
Expected Outcome
You have one clear message angle built around commission disappointment.

Step-by-Step Execution

  1. Open a new document called AESDR Trigger Angles. This will become your source file for all paid and organic messaging angles.
  2. Create a section named Paycheck Shock. Under it, write the exact moment: rep opens payroll or commission statement and realizes the money is nowhere near expected.
  3. Write the emotional facts in plain language: confusion, anger, distrust, shame, and the feeling of working hard without leverage.
  4. Draft three ad hooks. Keep each one short. Example formats: 'Your OTE is fiction if your process is chaos.' 'Clawbacks are not a growth plan.' 'If your paycheck keeps surprising you, your system is broken.'
  5. Draft one landing-page-adjacent sentence you can use later near pricing or the hero: 'The gap between expected money and actual money is usually process, not effort.'
  6. Save the document and label the angle with a simple code like TE-01 so you can track it later in ads and UTM naming.
  7. If you want to test the angle visually, write a simple creative brief: screenshot of payroll app, dark background, one brutal line of copy, CTA to diagnostic or course page.

Done When

  • At least three usable hooks exist.
  • The angle is tied to a real buyer moment, not generic pain.
  • The message can be reused in ads, email, and page copy.

Common Mistakes

  • Writing abstract slogans instead of buyer moments.
  • Overexplaining the angle in the hook itself.
  • Failing to save one master doc for all trigger angles.

Audit Tone against LinkedIn Cringe

Scrub out every phrase that makes the site sound like recycled sales-influencer content.

Owner
Founder
Systems / Tools
Entire landing page, optional grep/search in code editor
Dependencies
Current site copy in editable text.
Files / Places You’ll Touch
All HTML files for landing and course intro pages
Expected Outcome
The site sounds original and grounded.

Step-by-Step Execution

Files / Places You’ll Touch

Expected Outcome
The commission tracker appears inside Lesson 10.1 and works on desktop and mobile.

Step-by-Step Execution

TASK 02.07

Verify State-Saving Mechanics

Save learner progress so buyers can leave and come back without losing their place.

Owner
Backend Dev
Systems / Tools
Supabase, lesson app, auth state, progress table or JSON column
Dependencies
Supabase project and authentication exist first.
Files / Places You’ll Touch
App code; Supabase table definitions
Expected Outcome
Progress can be stored per user and restored when they return.

Step-by-Step Execution

  1. Decide what 'progress' means. Keep it simple at first: lesson completed yes or no, last lesson visited, and maybe form or checklist state for key activities.
  2. In Supabase, create a table such as course_progress with columns like id, user_id, lesson_key, completed, and updated_at. If you want flexible stored state, add a jsonb column named state_data.
  3. In your app, make sure the learner is authenticated before saving progress. Without a user id, the app cannot know whose progress it is.
  4. Add a save action in the lesson UI. The easiest starting point is a button like 'Mark Lesson Complete' that writes one row or updates one row in Supabase.
